# cp-scrub redaction benchmark

Scores PII/secret scrubbers against the [golden corpus](../corpus/) on the two axes
the research says decide adoption: **recall** (leak rate) and **over-redaction**
(false positives on must-survive strings — what actually drives abandonment).

```
uv run python benchmark/score.py
```

Runs `naive-regex`, `detect-secrets`, `presidio-vanilla`, and `cp-scrub` out of the
box; `gitleaks` and `trufflehog` are included automatically if their binaries are on
`PATH`. See [RESULTS.md](./RESULTS.md) for a sample run and interpretation.

## How scoring works

- The corpus stores engine-neutral ground truth: `sensitive` spans (canonical types)
  + `guard` substrings that must survive. Each tool is run on every case's raw input.
- **Recall** = sensitive spans *in the tool's scope* that it flagged. Secret scanners
  are scored on `secret/*`; Presidio on PII/NER; cp-scrub on everything.
- **Over-redaction** = must-survive `guard` strings the tool wrongly flagged (git
  SHAs, UUIDs, package names, …). Scored on every tool — nothing should flag these.

## Honesty / fairness notes (read before quoting numbers)

- **cp-scrub authored the corpus** → it is the *reference* that defines ground truth
  (≈100% recall by construction). It is labelled as such and is **not a fair entrant**.
  Its over-redaction number *is* fair (guards are independent of authorship) — and it
  is non-zero, which is the point: precision is hard for everyone.
- **gitleaks / trufflehog are repository secret scanners**, tuned for live, verifiable
  credentials in git history with allow-lists for known *example* keys. Scoring them on
  a synthetic conversation corpus is partly out-of-domain — low recall here reflects
  design intent (precision + verification), not pure inferiority. The honest takeaway
  is *"secret scanners are not conversation scrubbers,"* not *"tool X is bad."*
- **The corpus is synthetic/revoked-only** (hygiene: never a live credential), so it
  structurally cannot exercise trufflehog's live-verification strength → its recall
  here is a floor, not a verdict.
- Independence: the corpus was **not** generated by any scanner being ranked
  (SecretBench's contamination pitfall), so the *independent* tools' scores are fair.
